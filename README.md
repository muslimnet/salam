# Разные лицензии и что значит без лицензии

## По поводу репозиториев «Без лицензии»

Существует множество репозиториев, где владелец опубликовал код и релизы, но без надлежащей лицензии. 
Поскольку лицензия не предоставляется вместе с кодом, в настоящее время это репозитории без лицензии.

Если вы обнаружите программное обеспечение без лицензии, это, как правило, означает, что у вас нет разрешения от создателей программного обеспечения на использование, модификацию или распространение этого программного обеспечения.

По умолчанию авторское право устанавливает неразрешительный режим: как только произведение создано, оно автоматически защищается, и создатель сохраняет все исключительные права, если явно не указано иное. Это означает, что без лицензии пользователи юридически лишены возможности использовать, изменять, распространять или исполнять программное обеспечение. Таким образом, вариант «Без лицензии» представляет собой это состояние по умолчанию, неразрешительное, когда произведение остается под полным контролем создателя, без предоставления каких-либо разрешений другим лицам.

## Лицензии MIT или Apache

Разрешительные лицензии, такие как лицензии MIT или Apache, позволяют пользователям делать с программным обеспечением почти всё, включая создание коммерческих производных с закрытым исходным кодом, но обычно с правильными уведомлениями 
об использовании.

## Лицензии GPL

Лицензии GPL, гарантируют, что все модифицированные версии программного обеспечения остаются такими же свободными и открытыми, сохраняя эти свободы для всех последующих пользователей.


# Работаем

## Создание репозитория

GitHub позволяет добавить файл README одновременно с созданием нового репозитория.

## Создание ветки

Ветвление позволяет иметь разные версии репозитория одновременно.

По умолчанию в вашем репозитории есть одна ветка с именем main, которая считается основной. Вы можете создавать дополнительные ветки от main в вашем репозитории.

Ветвление полезно, когда вы хотите добавить новые функции в проект, не изменяя основной исходный код. 
Работа, проделанная в разных ветках, не отобразится в основной ветке, пока вы не выполните слияние.

Когда вы создаете ветку от main, вы делаете копию, или снимок, main в том виде, в котором она была на тот момент времени.

## Коммиты

Вы можете вносить и сохранять изменения в файлы вашего репозитория. На GitHub сохраненные изменения называются коммитами. Каждый коммит имеет связанное с ним сообщение, которое представляет собой описание, объясняющее, почему было внесено то или иное изменение. Сообщения коммитов фиксируют историю ваших изменений, чтобы другие участники могли понять, что вы сделали и почему.
