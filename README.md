# Разные лицензии и что значит без лицензии

Обычно текст лицензии помещают в файл с именем LICENSE.txt или LICENSE.md в корень репозитория. 

## По поводу репозиториев «Без лицензии»

Существуют репозитории, где владелец опубликовал код и релизы, но без лицензии. 
То есть это случай когда лицензия не предоставляется вместе с кодом и/или программой, произведением.

Если вы обнаружите программное обеспечение или его код без лицензии, это, как правило, означает, что у вас нет разрешения от авторов на использование, модификацию или распространение этого.

По умолчанию авторское право устанавливает неразрешительный режим: как только произведение создано, оно автоматически защищается, и автор сохраняет все исключительные права, если явно не указано иное. Это означает, что без лицензии пользователи юридически лишены возможности использовать, изменять, распространять или исполнять программное обеспечение. Таким образом, вариант «Без лицензии» представляет собой это состояние по умолчанию, неразрешительное, когда произведение остается под полным контролем автора, без предоставления каких-либо разрешений другим лицам.

То есть вы можете только смотреть, изучать и читать опубликованный код, чтобы, например, узнать возможности и функциональность программы, понять не совершает ли она неприемлимые для вас действия, узнать её структуру и используемые технологии, языки, комментарии.

Это в чём-то подобно просмотру исходного кода веб-страниц.

## Лицензии MIT или Apache

Разрешительные лицензии, такие как лицензии MIT или Apache, позволяют пользователям делать с программным обеспечением почти всё, включая создание коммерческих производных с закрытым исходным кодом, но обычно с правильными уведомлениями 
об использовании.

## Лицензии GPL

Лицензии GPL, гарантируют, что все модифицированные версии программного обеспечения остаются такими же свободными и открытыми, сохраняя эти свободы для всех последующих пользователей.


# Работаем

## Создание репозитория

GitHub позволяет добавить файл README одновременно с созданием нового репозитория.

## Создание ветки

Ветвление позволяет иметь разные версии репозитория одновременно.

По умолчанию в вашем репозитории есть одна основная ветка с именем main. 

Вы можете создавать дополнительные ветки от main в вашем репозитории.

Ветвление полезно, когда вы хотите добавить новые функции в проект, не изменяя основной исходный код. 
Работа, проделанная в разных ветках, не отобразится в основной ветке, пока вы не выполните слияние.

Когда вы создаете ветку от main, вы делаете копию main в том виде, в котором она была в тот момент времени.

## Коммиты

Вы можете вносить и сохранять изменения в файлы вашего репозитория. Сохраненные изменения называются коммитами. Каждый коммит имеет связанное с ним сообщение, которое объясняет изменение. Сообщения коммитов фиксируют историю изменений.
